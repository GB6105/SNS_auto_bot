# PRD — ADHD 플래너 SNS 브랜딩 자동화 에이전트 (MVP)

*인스타그램 + 스레드 · 반자동(승인제) 콘텐츠 파이프라인 · 작성일 2026-06-23*

---

## 1. 한 줄 요약

> ADHD 플래너의 핵심 메시지("흐릿한 목표 → 지금 할 첫 행동")를 콘텐츠로 변환해, **기획부터 게시까지 자동 생성**하고 사람은 **하루 한 번 텔레그램에서 승인만** 하는 SNS 브랜딩 에이전트.

**High-Level Concept:** 리소스 없이도 ADHD 커뮤니티 인지도·팔로워를 키우는 반자동 콘텐츠 공장.

---

## 2. 문제 (Problem)

제품(플래너)은 만들고 있으나, 이를 알릴 **콘텐츠 발행이 지속되지 않는다.**

1. **리소스 부재** — 콘텐츠 제작에 쓸 인력·시간이 거의 없다.
2. **지속성 실패** — 직접 만들면 초반에 지쳐 발행이 끊긴다(만들다 멈추는 ADHD 패턴).
3. **진정성 리스크** — 완전 무인 봇 느낌은 진정성에 민감한 ADHD 커뮤니티에서 신뢰를 잃는다.

**이번 MVP가 푸는 것:**
제작 리소스가 거의 없는 상태에서, *진정성을 지키면서 꾸준히* 콘텐츠를 발행하는 시스템을 만든다.

---

## 3. 핵심 전략 (Strategy)

| 구분 | 내용 |
|------|------|
| **목표** | ADHD 커뮤니티 내 인지도·팔로워 확보 (얼리어답터 풀 구축) |
| **방식** | "완전 무인"이 아니라 **"5분 승인제 반자동"** — 자동화 90%, 사람 개입 10% |
| **원칙** | 에이전트가 기획·카피·이미지까지 생성 → 사람은 텔레그램에서 "게시/수정/폐기"만 |

> 진정성 확보가 핵심 제약. 사람의 최종 승인 1탭이 봇 인상을 없애는 안전장치 역할을 한다.

---

## 4. 솔루션 (Solution) — 4개 모듈

```
[1. 기획·캘린더]  →  [2. 카피 생성]  →  [3. 이미지 생성]  →  [4. 승인]  →  [5. 게시·수집]
   주 1회 배치        Claude API         pencil.dev        텔레그램 1탭    n8n/스케줄러
```

1. **기획 모듈 (Brain)** — 4개 콘텐츠 기둥에서 주제를 뽑아 캘린더 자동 배치 (주 1회)
2. **카피 모듈 (Writer)** — 주제 → 인스타 캡션 + 스레드 글 동시 생성, 플랫폼별 톤 변주
3. **비주얼 모듈 (Designer)** — pencil.dev 디자인에 카피만 꽂는 슬롯 구조, 디자인은 매주 교체
4. **게시·운영 모듈 (Publisher)** — 스케줄 게시 + 댓글 응대 초안 + 성과 수집

## 5. 콘텐츠 기둥 (Content Pillars)

| 기둥 | 목적 | 효과 |
|------|------|------|
| **공감형** | ADHD 증상·상황 묘사 ("이거 내 얘기") | 저장·공유 유발 |
| **실전팁형** | 2분 시작법·복귀 기준 등 바로 쓰는 팁 | 유용성, 팔로우 동기 |
| **비포애프터형** | 흐릿한 할 일 → 정리된 첫 행동 예시 | 제품 가치 노출 |
| **빌드인퍼블릭형** | 제품 만드는 과정 공유 | 얼리어답터 모으기 |

---

## 6. 발행 리듬 (Cadence)

### 인스타그램 — 주 3회, 카드뉴스(이미지 + 캡션)

| 요일 | 기둥 | 비고 |
|------|------|------|
| 월 | 공감형 | 증상 묘사로 저장 유발 |
| 수 | 실전팁형 | 바로 쓰는 카드 |
| 금 | 비포애프터 / 빌드인퍼블릭 | 격주 교차 |

- 카드 구성: **5~7장** (저장률 최적)
- 1장 후킹 헤드라인 / 2~6장 한 장당 한 메시지 / 마지막 장 CTA

### 스레드 — 매일 1회, 텍스트

| 요일 | 톤 | 의도 |
|------|------|------|
| 월·목 | 진지한 글 | 실행기능 이야기·제품 철학 |
| 화·금 | 일상 + 가벼운 홍보 | "만들면서 겪은 것" 형태 |
| 수 | 질문형 | 댓글 = 팔로워 전환 |
| 토·일 | 짧은 공감 한 줄 | 부담 없는 노출 |

- 글당 280자 내외, 첫 문장에서 멈추게, 질문형은 답하기 쉬운 질문으로 마무리

---

## 7. 콘텐츠 생성 규칙 (Generation Rules)

### 인스타 카드뉴스
- **입력:** 기둥 + 주제 한 줄
- **출력:** 1장 후킹 헤드라인 → 2~6장 핵심(장당 한 메시지) → 마지막 CTA
- **디자인 규칙:** 텍스트 면적 20% 이내, 강한 색 대비, 1080×1080, 시선 흐름(헤드라인→비주얼→CTA)
- **캡션:** 첫 줄 후킹 + 본문 + 해시태그(#ADHD #성인ADHD #실행기능 등)

### 스레드 텍스트
- **입력:** 요일 톤
- **출력:** 280자 내외, 추상어 금지·구체 장면, 홍보는 "겪은 것" 형태, 질문형은 질문으로 종료

### 공통 톤 (PRD 플래너 철학 계승)
- 추상적 표현("열심히") 금지, 항상 결과물·구체 기준
- 자책 유발 금지, 복귀·재시작을 함께 제시

## 8. 가드레일 (Compliance & Safety)

자동 생성이라 사람 검수가 빠지는 만큼, 규칙을 **생성 단계(프롬프트)와 승인 단계(체크리스트)** 양쪽에 심는다.

### 출처·저작권
- 확인 안 된 구체적 통계는 넣지 않는다 (AI 환각 방지). 넣을 경우 검증된 출처 목록에서만.
- 통계·인용 출처는 마지막 카드 또는 캡션에 명시.
- 원문 복붙 금지 → 항상 자기 표현으로 재서술, 직접 인용은 짧게 + 출처.
- pencil.dev 디자인·폰트·이미지의 상업적 사용 라이선스 확인.

### 의학 정보
- 치료·완치 단정 금지. ADHD는 의료 주제 → 카드에 "의학적 조언 아님, 진단·치료는 전문가 상담" 취지 면책 삽입.

### 개인정보
- 커뮤니티 글·사연은 그대로 옮기지 않는다 → 여러 사례를 익명의 일반 상황으로 재구성.
- 캡처·닉네임·특정 가능한 디테일 금지.
- 랜딩 입력 데이터(사용자의 사적 정보)는 동의 없이 콘텐츠화 금지.
- 빌드인퍼블릭에서 본인 실명·위치·계정 등 식별 정보 노출 점검.

### 게시 전 자동 체크리스트 (승인 단계)
1. 확인 안 된 통계가 있는가?
2. 남의 사연·캡처가 있는가?
3. 의학적 단정이 있는가?
4. 출처 표기가 필요한 이미지인가?

## 9. 기술 스택 & 실행 환경

### 모듈별 도구

| 모듈 | 도구 | 비고 |
|------|------|------|
| 기획·캘린더 | 스프레드시트 / Notion + Claude API | 주 1회 배치 |
| 카피 생성 | Claude API | 시스템 프롬프트에 톤·가드레일 고정 |
| 이미지 생성 | pencil.dev | 디자인 매주 교체, 카피 슬롯 구조 |
| 승인 | **텔레그램 봇** | 하루 1회 알림, "게시/수정/폐기" 1탭 |
| 게시·수집 | n8n + IG Graph API + Threads API | 스케줄·도달·댓글 수집 |

### 실행 환경 (호스팅) — 결정

오케스트레이션은 **n8n**으로 하고, 호스팅은 아래 두 안 중 택일.

| 옵션 | 비용 | 세팅 | 적합성 |
|------|------|------|--------|
| **n8n 클라우드 Starter (1차 선택)** | 월 \$20~24 (2,500 실행) | 0분, 바로 시작 | "오늘 완성" 목표에 최적, 서버관리 불필요 |
| 자체 호스팅 (PikaPods/InstaPods) | 월 \$3~7 (실행 무제한) | 1~4시간 + 유지보수 월 1~2h | 비용 절약, 손에 익은 뒤 이전 |

> **사용량 추정:** 스레드 일 1회(~30/월) + 인스타 주 3회(~13) + 캘린더 주 1회(~4) + 게시(~43) + 댓글 초안(~50~100) = **월 200~300 실행**. Starter 한도(2,500)에 한참 못 미쳐 여유가 큼. n8n은 단계 수가 아니라 워크플로우 1회 실행을 1건으로 카운트(성공 실행만 집계).
>
> **전략:** 1차는 Starter로 빠르게 띄우고(14일 무료체험 가능), 손에 익으면 자체 호스팅으로 이전해 비용 절감.

### 플랫폼 API 제약 (확인 필요)
- 인스타: **비즈니스/크리에이터 계정** 필수 + 페이스북 페이지 연결, 앱 심사 권한별 2~4주, 게시 한도 24h당 25건.
- 스레드: 인스타 API는 텍스트 전용 미지원 → **별도 Threads API** 사용.
- 심사 지연 시 공백은 수동 게시로 메움.

## 10. 구동 방식 (How It Runs) — 어디서·언제·어떻게

에이전트는 상시 구동 프로그램이 아니라 **정해진 시각에 깨어나 할 일만 하고 다시 자는** 구조.

### 어디서 (Where)
n8n 클라우드 Starter 인스턴스 위에서 워크플로우가 구동. 별도 서버 관리 없음.

### 언제 (When) — 두 종류 트리거

**시간 기반 (Cron) — 생성을 자동으로 깨움**

| 시점 | 동작 |
|------|------|
| 매주 일요일 밤 | 다음 주 주제 캘린더 생성 + 카피 일괄 생성 |
| 매일 오전 8시 | 그날 스레드 글 생성 → 승인 알림 |
| 월·수·금 | 인스타 카드뉴스 생성 → 승인 알림 |

**이벤트 기반 (Webhook) — 사람·외부 반응에 반응**

| 트리거 | 동작 |
|--------|------|
| 텔레그램 "게시" 버튼 클릭 | 실제 게시 실행 |
| 게시물 댓글 발생 | 답변 초안 생성 → 텔레그램 알림 |

### 어떻게 (How) — 실행 흐름 (인스타 예시)

```
[Cron: 수요일 오전 8시]
   ↓
1. 캘린더에서 오늘 주제 읽기
   ↓
2. Claude API로 카드뉴스 카피 + 캡션 생성 (가드레일 프롬프트)
   ↓
3. pencil.dev 디자인 슬롯에 카피 꽂아 이미지 생성
   ↓
4. 자동 체크리스트 표시 → 텔레그램으로 미리보기 알림
   ↓
   ⏸ 멈춤 — 사람의 1탭 대기
   ↓
[Webhook: "게시" 버튼 클릭]
   ↓
5. IG Graph API 호출 (미디어 컨테이너 생성 → 게시)
   ↓
6. 게시 결과·성과 수집
```

> 스레드도 동일 패턴, 5번만 Threads API로 교체.
> **핵심:** 4번과 5번 사이의 "멈춤"이 PRD의 5분 승인제를 구현하는 지점. 생성은 완전 자동, 게시는 사람의 1탭이 트리거.

## 11. MVP 범위

### In Scope
- 4개 콘텐츠 기둥 + 2주치 주제 캘린더
- 인스타 카드뉴스 / 스레드 요일별 생성 프롬프트(가드레일 포함)
- pencil.dev 디자인용 카피 슬롯 구조
- n8n 워크플로우(Cron + Webhook) + 텔레그램 승인 봇
- 게시 후 성과 수집(도달·저장·댓글)

### Out of Scope
- 완전 무인 게시(승인 단계 항상 유지)
- 유료 광고 집행
- 멀티 계정 관리
- 댓글 완전 자동 응대(초안까지만, 게시는 승인)

| 단계 | 지표 | 의미 |
|------|------|------|
| Acquisition | 도달·노출 | 콘텐츠가 커뮤니티에 닿는가 |

| **핵심** | 팔로워 증가 + 프로필 → 랜딩 클릭 | 얼리어답터 풀이 쌓이는가 |

## 13. 위협 / 리스크

- **진정성:** 자동 생성 티가 나면 커뮤니티 신뢰 하락 → 승인 단계·톤 변주로 완화.
- **API 문턱:** 계정 전환·앱 심사 지연 → 심사 전 수동 게시로 공백 메움.
- **콘텐츠 고갈:** 주제 소진 → 캘린더 주 1회 보충 + 커뮤니티 반응에서 소재 추출.
- **가드레일 누락:** 자동화 사고 → 생성·승인 양쪽 이중 방어.
- **실행 한도/비용:** 댓글 폭증 등으로 실행 급증 → Starter 한도(2,500) 내 여유 크나, 초과 시 자체 호스팅 이전.
