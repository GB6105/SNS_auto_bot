# PM Analysis

`enriched-plan.md`를 에픽으로 그룹화하고 사용자 스토리화한다. 우선순위(P0/P1/P2)는 PM 주권. T-shirt 추정(XS~XL)과 의존성을 부여한다. 단일 운영자(승인자) = "운영자", 콘텐츠 수신자 = "팔로워".

## 에픽

- **EPIC-1 콘텐츠 기획·캘린더** — 4기둥에서 주제를 뽑아 2주치 캘린더를 cadence 규칙대로 배치.
- **EPIC-2 카피 생성(가드레일 내장)** — 주제 → IG 카드뉴스 카피+캡션, 스레드 텍스트를 플랫폼/요일 톤으로 생성.
- **EPIC-3 비주얼 슬롯(pencil.dev 계약)** — 카피를 디자인 슬롯에 꽂는 범용 슬롯 JSON 계약 + 어댑터.
- **EPIC-4 승인·게시(텔레그램/IG/Threads)** — 승인 게이트 + 게시 어댑터 + 성과 수집.
- **EPIC-5 오케스트레이션(n8n + Cron/Webhook)** — 생성→체크리스트→승인→게시 파이프라인 구동.

## 사용자 스토리

### EPIC-1 콘텐츠 기획·캘린더

- [ ] **US-1 (P0, M)** 2주치 콘텐츠 캘린더 생성
  - As a 운영자 / I want 4기둥과 cadence 규칙으로 2주치 캘린더가 자동 배치되기를 / So that 매주 무엇을 만들지 고민하지 않는다.
  - 수용기준: ① 시작일 입력 시 14일치 슬롯 생성 ② IG=월·수·금에 기둥 매핑(금은 비포애프터/빌드인퍼블릭 격주 교차) ③ 스레드=매일 요일 톤 매핑 ④ 각 항목 {date, platform, pillar/tone, status:"planned"} ⑤ 결정적(같은 시작일→같은 캘린더).

- [ ] **US-2 (P1, S)** 기둥별 주제 풀에서 주제 1줄 선정
  - As a 운영자 / I want 각 캘린더 슬롯에 기둥에 맞는 주제 한 줄이 채워지기를 / So that 카피 생성 입력이 준비된다.
  - 수용기준: ① 기둥별 주제 후보에서 미사용 우선 선택 ② 슬롯에 topic 필드 주입 ③ 중복 회피.

### EPIC-2 카피 생성

- [ ] **US-3 (P0, M)** IG 카드뉴스 카피+캡션 생성
  - As a 운영자 / I want 기둥+주제 입력으로 5~7장 카드 카피와 캡션이 생성되기를 / So that 디자인에 꽂을 텍스트가 준비된다.
  - 수용기준: ① 1장 후킹 헤드라인 + 2~6장 한 장 한 메시지 + 마지막 CTA ② 캡션=첫줄 후킹+본문+해시태그 ③ 가드레일 프롬프트 적용 ④ 카드 수 5~7 범위 ⑤ 구조화 출력(JSON).

- [ ] **US-4 (P0, M)** 스레드 텍스트 생성(요일 톤)
  - As a 운영자 / I want 요일 톤에 맞는 280자 내외 스레드 글이 생성되기를 / So that 매일 텍스트 발행이 끊기지 않는다.
  - 수용기준: ① 요일→톤 매핑 적용(진지/일상홍보/질문형/공감) ② 280자 내외 ③ 추상어 금지·구체장면 ④ 질문형은 질문으로 종료 ⑤ 가드레일 적용.

- [ ] **US-5 (P0, M)** Claude API 클라이언트 + 오프라인 stub
  - As a 개발자 / I want 카피 생성이 Claude API로 동작하되 키가 없으면 결정적 stub로 동작하기를 / So that 외부 키 없이도 개발·테스트가 된다.
  - 수용기준: ① `ANTHROPIC_API_KEY` 있으면 실호출 ② 없으면 결정적 stub 응답 ③ 시스템 프롬프트에 톤·가드레일 고정 ④ 모델/온도 설정 가능.

### EPIC-2/4 가드레일

- [ ] **US-6 (P0, M)** 게시 전 자동 체크리스트(규칙 스캐너)
  - As a 운영자 / I want 생성 콘텐츠가 게시 전 4종 체크를 자동 통과/경고되기를 / So that 자동화 사고(통계·사연·의학·출처)를 막는다.
  - 수용기준: ① 미확인 통계 패턴 탐지 ② 남의 사연·캡처 신호 탐지 ③ 의학적 단정 탐지 ④ 출처 필요 플래그 ⑤ {check, status:pass|warn, evidence[]} 리포트 ⑥ 경고 1건↑이면 승인 카드에 노출.

### EPIC-3 비주얼 슬롯

- [ ] **US-7 (P1, M)** pencil.dev 카피 슬롯 계약 + 어댑터 인터페이스
  - As a 개발자 / I want 카피를 `{slotId, role, text, maxChars}` 범용 슬롯 JSON으로 매핑하고 어댑터 인터페이스로 추상화하기를 / So that pencil.dev 연결 방식이 정해지면 어댑터만 갈아끼운다.
  - 수용기준: ① 카드/캡션 → 슬롯 배열 변환 ② maxChars 초과 시 경고 ③ `DesignAdapter` 인터페이스 정의 ④ 미연결 시 슬롯 JSON을 파일로 출력하는 기본 어댑터.

### EPIC-4 승인·게시

- [ ] **US-8 (P0, S)** 승인 게이트(상태 머신)
  - As a 운영자 / I want 생성물이 "게시/수정/폐기" 결정 전까지 멈추기를 / So that 5분 승인제가 구현된다.
  - 수용기준: ① 상태 planned→generated→awaiting_approval→(approved|revise|discarded)→published ② 승인 없이는 publish 불가 ③ 24h 무응답→discarded(자동 게시 금지) ④ 상태 전이 검증.

- [ ] **US-9 (P1, M)** 텔레그램 승인 봇 어댑터(인터페이스+stub)
  - As a 운영자 / I want 미리보기+버튼 알림을 받고 1탭으로 결정하기를 / So that 하루 한 번 승인만 한다.
  - 수용기준: ① 미리보기 메시지 포맷(카피+체크리스트 경고) ② inline 버튼(게시/수정/폐기) ③ `TELEGRAM_BOT_TOKEN` env ④ 토큰 없으면 콘솔 stub. (외부 토큰 필요 → 어댑터까지)

- [ ] **US-10 (P1, M)** IG/Threads 게시 어댑터(인터페이스+stub)
  - As a 운영자 / I want 승인 후 IG 카드/스레드 글이 게시되기를 / So that 발행이 완료된다.
  - 수용기준: ① `Publisher` 인터페이스(IG Graph: 컨테이너 생성→게시 / Threads: 텍스트) ② 자격증명 env 주입 ③ 미설정 시 dry-run stub ④ 게시 결과 객체 반환. (외부 심사/자격증명 필요 → 어댑터까지)

- [ ] **US-11 (P2, S)** 성과 수집(도달·저장·댓글)
  - As a 운영자 / I want 게시물 성과가 수집되기를 / So that 지표를 본다.
  - 수용기준: ① 게시 id로 인사이트 조회 인터페이스 ② {reach, saves, comments} 정규화 ③ 미설정 시 stub. (외부 API 필요)

### EPIC-5 오케스트레이션

- [ ] **US-12 (P1, M)** 로컬 파이프라인 오케스트레이터 + CLI
  - As a 운영자 / I want 캘린더→카피→슬롯→체크리스트→승인대기 미리보기를 한 번에 실행하기를 / So that 외부 설정 없이 전체 흐름을 확인한다.
  - 수용기준: ① 날짜 입력→그날 항목 생성·체크·미리보기 산출 ② JSON/콘솔 출력 ③ 결정적 ④ 승인 게이트에서 멈춤.

- [ ] **US-13 (P2, M)** n8n 워크플로우 블루프린트(Cron+Webhook)
  - As a 운영자 / I want n8n에 임포트할 워크플로우 JSON 설계도가 있기를 / So that 호스팅 후 바로 구축한다.
  - 수용기준: ① Cron(일요일/매일8시/월수금) 노드 ② 텔레그램 승인 Webhook 노드 ③ 댓글 Webhook ④ 생성→체크→알림→게시 흐름 주석. (n8n 클라우드 필요 → 설계도 JSON까지)
