# ADHD 플래너 SNS 브랜딩 자동화 에이전트 — 제품 요구사항 문서 (PRD)

> 생성: prd-scaffold-v0 파이프라인 · 버전 v0 · 2026-06-23

## 1. 제품 개요
ADHD 플래너 제품의 SNS 브랜딩을 자동화하는 **반자동(승인제) 콘텐츠 에이전트**. 플래너의 메시지("흐릿한 목표 → 지금 할 첫 행동")를 인스타그램 카드뉴스와 스레드 텍스트로 변환해 기획·카피·이미지까지 자동 생성하고, 사람은 하루 한 번 텔레그램에서 "게시/수정/폐기" 1탭만 한다. 자동화 90% / 사람 개입 10%의 "5분 승인제 반자동"으로, 진정성을 지키며 끊김 없는 발행을 목표한다.

## 2. 문제 정의
플래너 제품은 만들고 있으나 이를 알릴 콘텐츠 발행이 지속되지 않는다 — ① 제작 리소스 부재, ② 직접 만들면 지쳐 끊기는 지속성 실패(ADHD 패턴), ③ 완전 무인 봇은 진정성에 민감한 ADHD 커뮤니티 신뢰를 잃는 리스크. MVP는 "리소스 거의 없이, 진정성을 지키면서 꾸준히" 발행하는 시스템을 만든다.

## 3. 타겟 사용자
- **운영자(1인, 승인자):** ADHD 본인일 가능성이 큼. 하루 한 번 1탭 승인만 하고 싶다. 승인 피로·무응답에 취약.
- **팔로워(ADHD 커뮤니티):** "이거 내 얘기" 공감 → 저장/팔로우 → 프로필 → 랜딩 클릭으로 이어지는 얼리어답터 풀.

## 4. 제품 목표
- ADHD 커뮤니티 내 인지도·팔로워 확보(얼리어답터 풀 구축).
- 진정성 유지(사람 최종 승인 1탭이 봇 인상을 제거하는 안전장치).
- 발행 지속(시스템이 안 끊김) — 외부 키 없이도 전체 흐름이 도는 개발 가능 상태 확보.

## 5. 핵심 기능
1. **기획·캘린더** — 4기둥(공감/실전팁/비포애프터/빌드인퍼블릭)에서 주제를 뽑아 2주치 캘린더를 cadence 규칙대로 배치.
2. **카피 생성(가드레일 내장)** — IG 카드뉴스(5~7장)+캡션, 요일 톤 스레드(280자) 생성. Claude API + 무키 오프라인 stub.
3. **게시 전 자동 체크리스트** — 미확인 통계/남의 사연·캡처/의학 단정/출처 필요를 규칙 스캔(차단 아닌 경고 + 사람 2중방어).
4. **비주얼 슬롯(pencil.dev 계약)** — 카피를 role 기반 범용 슬롯 JSON으로 매핑 + DesignAdapter 인터페이스.
5. **승인·게시** — 상태머신 승인 게이트(무응답→리마인드→폐기, 자동게시 금지), 텔레그램/IG/Threads 어댑터(stub).
6. **오케스트레이션** — 로컬 CLI 파이프라인 + n8n(Cron+Webhook) 블루프린트.

## 6. 사용자 스토리

> 모든 스토리는 `passes:false`로 시작하며 구현 시 true로 전환된다.

### P0 (필수) — 외부 설정 없이 전부 구현 가능
- **US-1 2주치 캘린더 생성** — 시작일 입력 시 14일치 슬롯을 IG(월·수·금, 금은 비포애프터/빌드인퍼블릭 격주)·스레드(매일 요일톤)로 결정적 배치.
- **US-3 IG 카드뉴스 카피+캡션** — 1장 후킹+2~6장 한장한메시지+마지막 CTA, 캡션(후킹+본문+해시태그). 후처리 검증(카드수 5~7 클램프).
- **US-4 스레드 텍스트(요일 톤)** — 진지/일상홍보/질문형/공감 톤, 280자 내외 트림, 질문형은 질문 종료.
- **US-5 Claude 클라이언트 + 오프라인 stub** — 키 있으면 실호출, 없으면 결정적 stub. 모델 env 외부화(기본 override 가능).
- **US-6 게시 전 자동 체크리스트** — 4종 규칙 스캐너, {check,status:pass|warn,evidence[]}, warn은 정보(차단 아님).
- **US-8 승인 게이트(상태머신)** — planned→generated→awaiting_approval→(approved|revise|discarded)→published, 무응답 리마인드 후 24h 폐기, 승인 없이 게시 불가.

### P1 (중요)
- **US-2 기둥별 주제 풀 선정** — 미사용 우선 선택, 슬롯에 topic 주입, 중복 회피.
- **US-7 pencil.dev 슬롯 계약 + 어댑터** — role(headline/body/cta/disclaimer) 기반 슬롯 + slotId 매핑 분리, DesignAdapter, 기본=파일 출력 어댑터.
- **US-9 텔레그램 승인 봇 어댑터** — 미리보기+inline 버튼 포맷, 토큰 없으면 콘솔 stub. *(외부 토큰 필요 → 어댑터까지)*
- **US-10 IG/Threads 게시 어댑터** — IG 컨테이너→publish 2단계, Threads 텍스트, dry-run stub. *(앱심사·자격증명 필요 → 어댑터까지)*
- **US-12 로컬 파이프라인 + CLI** — 날짜 입력→캘린더·카피·슬롯·체크리스트·승인대기 미리보기를 한 번에, 결정적, 승인 게이트에서 멈춤.

### P2 (선택)
- **US-11 성과 수집** — {reach,saves,comments} 정규화 인터페이스, 미설정 시 stub. *(외부 API)*
- **US-13 n8n 워크플로우 블루프린트** — Cron(일요일/매일8시/월수금)+승인/댓글 Webhook 참조 JSON. *(n8n 호스팅 후 자격증명 연결)*

## 7. 기술 스택

| 항목 | 선택 | 비고 |
|---|---|---|
| 프레임워크 | Node 20+ (ESM) | 런타임 의존 0(코어), 모든 외부I/O 인터페이스+stub |
| 언어 | TypeScript | n8n/IG Graph/Threads 생태계 정합 |
| 테스트 | `npm test` | `tsc --noEmit && node --test` |
| 타입체크 | `npm run typecheck` | `tsc --noEmit` |
| 린트 | `npm run lint` | `tsc --noEmit`(ESLint 미도입, 의존 최소화) |

부가 도구: Claude API(카피, 모델 env), pencil.dev(이미지 슬롯, 계약만), 텔레그램 봇(승인), n8n(오케스트레이션), IG Graph API + Threads API(게시·수집) — 실연결은 외부 설정.

## 8. UI · UX
- **승인 카드:** 5초 내 판단 가능 — 헤드라인+장수, 캡션 첫 줄, 체크리스트 경고만 강조, 본문 접기. ⚠️+텍스트 라벨 병기(색만으로 구분 금지).
- **무응답 기본 = 폐기**(게시 아님) + 만료 전 리마인드 1회. [폐기]는 확인 1스텝, [수정]은 재생성(상한).
- **결정적 미리보기**로 시스템 신뢰 확보.
- **콘텐츠 접근성:** 카드 텍스트 면적 20% 이내·강한 색대비, 한 카드 한 메시지, 추상어 금지, "의학적 조언 아님" 면책 일관 위치.

## 9. 성공 지표

**주요 KPI**
- 팔로워 증가 + 프로필→랜딩 클릭(핵심: 얼리어답터 풀 적립).
- 발행 지속률(주 목표 대비 실제 게시) — 시스템이 안 끊기는가.

**보조 지표**
- 도달·노출(Acquisition), 저장·공유율(Activation), 댓글.

## 10. 오픈 이슈
1. **pencil.dev 연결 방식 미확정** — 범용 슬롯 JSON 계약 + DesignAdapter로 추상화. 실연결(API/export/수동)은 사용자 확인 후. *(현재 결정: 범용 슬롯 JSON)*
2. **IG 비즈 계정 + 앱 심사(2~4주)** — 그 사이 수동 게시(카피·슬롯 export)로 공백 메움.
3. **승인 만료/리마인드 정확값** — 기본 리마인드 +12h, 폐기 24h(운영 설정).
4. **모델 비용** — 카피는 sonnet 기본 권장 옵션, opus override.
5. **n8n 호스팅 택일** — 1차 Starter, 이후 자체 호스팅 이전.
