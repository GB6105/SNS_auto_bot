# User Analysis

`enriched-plan.md` + `pm-analysis.md` 기반 사용자 저니·UX 수용기준·마찰지점·접근성을 평가한다. 두 사용자: **운영자**(승인자, 1인·ADHD 본인일 가능성 큼)와 **팔로워**(ADHD 커뮤니티, 콘텐츠 수신자).

## 사용자 저니

### 첫 사용 (운영자 온보딩)
1. 저장소 클론 → `npm install` → `npm test` 통과 확인(외부 키 없이 동작).
2. CLI로 `생성 미리보기` 실행 → 캘린더·카피·체크리스트·슬롯 JSON을 콘솔/파일로 확인(stub 모드).
3. `ANTHROPIC_API_KEY` 설정 → 실제 카피 품질 확인.
4. (외부 설정 단계) 텔레그램 토큰/IG·Threads 자격증명/n8n 호스팅 연결.

### 일상 사용 (운영자, 하루 1회)
1. 오전 8시 — 텔레그램에 그날 스레드 미리보기 + 체크리스트 경고 도착.
2. (월·수·금) 인스타 카드뉴스 미리보기 도착.
3. 미리보기를 5초 훑고 **[게시] / [수정] / [폐기]** 1탭.
4. 게시 선택 → 자동 발행. 무응답 → 24h 후 자동 폐기(자동 게시 안 됨).
5. 댓글 발생 시 답변 초안 알림 → 검토 후 게시.

### 팔로워 저니(수신)
- 피드/스레드에서 "이거 내 얘기" 공감 → 저장/팔로우 → 프로필 → 랜딩 클릭.

## UX 수용 기준
- **승인 카드 가독성:** 미리보기는 5초 내 판단 가능해야 — 카드 헤드라인+장수, 캡션 첫 줄, 체크리스트 경고만 강조. 전체 본문은 접기/펼치기.
- **경고 우선 노출:** 체크리스트 warn은 미리보기 상단에 눈에 띄게(예: ⚠️ 미확인 통계). pass-only면 "검수 통과" 한 줄.
- **결정의 비가역성 안전장치:** [폐기]는 되돌릴 수 없으므로 확인 1스텝(또는 일정시간 내 취소). [수정]은 재생성 트리거.
- **무응답 기본 안전:** 무응답의 기본 결과는 "게시"가 아니라 "폐기" — 진정성·사고 방지.
- **결정적 미리보기:** 같은 입력은 같은 미리보기 → 운영자가 시스템을 신뢰.

## 마찰 지점(Friction)
1. **승인 피로:** 매일 알림이 부담 → 미리보기를 짧게, 경고 없으면 1탭으로 끝. ADHD 운영자 자신이 끊기지 않도록 알림은 하루 1묶음.
2. **수정 루프 모호:** [수정] 눌렀을 때 무엇이 바뀌는지 불명확 → "재생성됨(변주 N회)" 표시, 무한 재생성 방지(상한).
3. **체크리스트 오탐:** 규칙 스캐너가 정상 콘텐츠를 warn 처리하면 신뢰 하락 → warn은 차단이 아닌 정보. 근거(매칭 텍스트) 함께 표시해 운영자가 즉시 판단.
4. **외부 설정 벽:** IG 앱 심사 2~4주 → 그 사이 "수동 게시" 안내 + 슬롯 JSON/카피 export로 손게시 지원.
5. **첫 실행 진입장벽:** 키 없이도 전부 돌아가야 함(stub) → 온보딩 마찰 최소.

## 접근성 (Accessibility)
- **콘텐츠 산출물 측:** IG 카드 텍스트 면적 20% 이내·강한 색대비(원문 규칙) → 저시력/주의분산 사용자 가독. 카드 캡션에 카드 텍스트를 옮겨 적어(이미지 내 텍스트 대체) 스크린리더·SEO 대응 권장.
- **ADHD 인지 접근성:** 한 카드 한 메시지, 추상어 금지·구체 장면 → 인지 부하 감소(제품 철학과 일치).
- **운영 도구 측:** CLI/텔레그램 텍스트 인터페이스 → 저사양·모바일에서 접근 가능. 색만으로 경고 구분 금지(⚠️ + 텍스트 라벨 병기).
- **의료 면책 가시성:** "의학적 조언 아님" 면책을 카드/캡션에 일관 위치로 — 신뢰·안전.
