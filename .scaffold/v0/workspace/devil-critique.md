# Devil's Advocate Critique

이전 모든 `*.md`를 읽고 5유형(가정/범위/모순/타당성/가치)으로 정확히 챌린지한다. 전면 거부·신규 스토리 작성은 하지 않는다. 각 항목: 유형 + 대상 + 근거 + 제안.

## 챌린지 (7개)

1. **[타당성] US-3/US-4 — LLM 구조화 출력의 불안정성**
   - 대상: 카드뉴스/스레드 카피 JSON 생성.
   - 근거: LLM이 카드 수(5~7)·280자·"질문으로 종료" 같은 형식 제약을 항상 지키지 않는다. 무키 stub는 통과해도 실 API에서 깨질 수 있다.
   - 제안: 생성 후 **결정적 후처리 검증·보정 계층**(카드 수 클램프, 280자 트림, 질문형 종료 보장)을 US-3/US-4 수용기준에 포함하고 테스트로 잠근다.

2. **[가정] US-6 — 규칙기반 가드레일의 한계(오탐·미탐)**
   - 대상: 통계·사연·의학·출처 정규식 스캐너.
   - 근거: 정규식은 의학적 단정("ADHD는 치료된다")은 잡아도 우회 표현("싹 낫는대요")은 놓치고, 정상 통계 인용을 오탐한다. enriched의 "환각 방지"를 규칙만으로 보장한다는 가정은 약하다.
   - 제안: 가드레일을 **차단이 아닌 경고(warn)** 로 두고 사람 승인과 2중 방어로 명시(agent-book §가드레일 양쪽). LLM 자체검수(self-critique) 패스는 P2 확장으로 분리. 오탐 수용·근거표시는 user-analysis와 정합.

3. **[모순] 무응답 기본값 — "지속 발행" vs "무응답 폐기"**
   - 대상: US-8 24h 무응답→폐기 vs §핵심지표 "발행 지속률".
   - 근거: ADHD 운영자가 승인을 자주 놓치면(바로 그 페르소나의 특성) 무응답 폐기가 누적돼 발행이 끊긴다 — 시스템이 풀려던 "지속성 실패"를 되살린다.
   - 제안: 폐기 대신 **리마인드 1회 후 만료**(예: +12h 리마인드 → 24h 폐기), 그리고 "오늘 게시 안 함"이 누적되면 알림. 자동 게시는 여전히 금지. US-8 수용기준에 리마인드 훅 반영(설정값).

4. **[범위] US-13 n8n 블루프린트 — "설계도"의 검증 불가**
   - 대상: 임포트용 워크플로우 JSON.
   - 근거: 실행 가능한 n8n JSON은 노드 버전/자격증명 참조에 민감해 "설계도"가 임포트 시 깨질 위험. 코드 테스트로 검증 불가(외부 런타임).
   - 제안: US-13을 **P2 + "참조용 블루프린트(주석 포함, 임포트 후 자격증명 연결 필요)"** 로 범위를 명확히. JSON 유효성(파싱)만 로컬 검증, 실행은 호스팅 후 수동. 이미 P2이므로 ESCALATE 불필요.

5. **[가치] US-11 성과 수집 — MVP 가치 대비 우선순위**
   - 대상: 도달·저장·댓글 수집.
   - 근거: 외부 API 권한 의존이 크고, MVP 1차 목표(발행 지속·진정성)에 직접 기여하지 않는다. 측정은 게시가 돌기 시작한 뒤 가치가 생긴다.
   - 제안: 현행 P2 유지가 적절(이미 후순위). 인터페이스만 두고 실수집은 외부 설정 이후로. (제거 아님 — 지표는 §12에 명시된 핵심 가치)

6. **[가정] pencil.dev "매주 디자인 교체" — 슬롯 안정성**
   - 대상: US-7 슬롯 계약, enriched 가정 4.
   - 근거: 디자인을 매주 교체하면 슬롯 id/구조도 매주 바뀔 수 있어, 고정 슬롯 계약 가정이 깨진다.
   - 제안: 슬롯 계약을 **role 기반**(`headline`/`body`/`cta`/`disclaimer`)으로 정의하고 slotId는 매핑 테이블로 분리 → 디자인 교체 시 매핑만 갱신. US-7 수용기준에 role enum 명시.

7. **[타당성] US-5 — "최신 Claude 모델" 하드코딩 위험**
   - 대상: 기본 모델 `claude-opus-4-8`.
   - 근거: 모델 id를 코드에 박으면 더 이상 비용/가용성 변화에 약하다. 비용 민감 운영(ADHD 1인)에 opus 기본은 과할 수 있다.
   - 제안: 모델을 **env/설정으로 외부화**(기본값은 두되 override 가능), 카피 같은 정형 작업은 sonnet 기본 권장 옵션 제공. US-5 수용기준 "모델 설정 가능"과 정합.

## 요약
미해결 분쟁 없음. 모든 챌린지는 기존 스토리의 **수용기준 보강**으로 해소 가능(신규 기능 발명 아님). 핵심: ①생성 후처리 검증(US-3/4), ②가드레일=경고+사람 2중방어(US-6), ③무응답 리마인드 후 폐기(US-8), ④슬롯 role 기반(US-7), ⑤모델 외부화(US-5). P0 스토리에 대한 blocker 챌린지 없음 → SlopChecker ESCALATE 불요.
